# VEP 2 - Standard Format of vindex.toml

## Abstract

This VEP specify the standard format of `vindex.toml`, the project configuration file of all Vix project.

`vindex.toml` is the standard configuration file of all Vix project, declaring specification data of the project, such as project meta data, dependencies, etc.

By specifying a specific format of `vindex.toml`, developers can not only write accurate configuration file quickly without any ambiguousness and manage there projects easily, but also use both offical and third-party package managers freely. At the same time, community contributors are able to develop their package managers with much freedom as long as they follow this standard.


## Motivation

As Vix develops, more and more third-party tools and libraries had come out. If there were no standard configuration file for these tools and libraries, some guesses and thinking may be required for users to figure out what is the tool/library for, and package managers aren't able to resolve dependencies in a simpe way. To sole this problem, `vindex.toml` rises in response to the proper time and conditions.


## Rationale

As a system-level programming language with a rich community supporing, managing big projects and using dependencies is unavoidable. To discribe project information in a simple way, we introduce a configuration file called `vindex.toml` to make this easier.

## Specification

The `[project]` table is the only required section in `vindex.toml`. It describes the project metadata and its dependencies.

### `[project]` Fields

| Field       | Type     | Required | Default    | Description                                                                 |
|-------------|----------|----------|------------|-----------------------------------------------------------------------------|
| `name`      | string   | yes      | ÔÇö          | Project name, used as the default output filename for compiled artifacts.   |
| `version`   | string   | no       | ÔÇö          | Project version.                                                            |
| `entrypoint`| string   | no       | `"main.vix"`| Entry file for compiled projects. **Third-party libraries must not set this**; the library entry must be `main.vix`. |
| `deps`      | array    | no       | `[]`       | List of dependency packages. Each element is a dependency specifier (supports all shorthand formats of the `add` command). |

### Dependency Specifier Formats

Each element in `deps` uses one of the following shorthand formats, which the `add` command resolves to a full URL:

| Format               | Example                     | Resolved URL                          |
|----------------------|-----------------------------|---------------------------------------|
| Bare name            | `vnet`                      | `github.com/vixlang/vlib-vnet`        |
| `user.repo`          | `fexcode.vnet`              | `github.com/fexcode/vnet`             |
| `user.repo@branch`   | `fexcode.vnet@master`       | `github.com/fexcode/vnet` #master     |
| `host:user.repo`     | `gitee.com:fexcode.vnet`    | `gitee.com/fexcode/vnet`              |
| `@user.repo`         | `@fexcode.vnet`             | `gitee.com/fexcode/vnet`              |

### Example

```toml
[project]
name = "webapp"
version = "1.0.0"
entrypoint = "app/main.vix"
deps = [
    "vnet@master",
    "gitee.com:fexcode.vlib-html",
    "vlib-json",
]
```
